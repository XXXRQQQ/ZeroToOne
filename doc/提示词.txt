你是一个数据大屏专家。请为项目 ZeroToOne（从零到一数据大屏）创建一个完整、漂亮、可运行的数据可视化大屏页面。

## 技术栈
- Vue 3 (Composition API) + TypeScript
- Vite 5 构建工具
- Pinia 状态管理
- ECharts 5 图表库
- SCSS 样式
- 大屏适配方案：使用 rem 或 scale 缩放方案，适配 1920×1080 主流分辨率

## 项目要求

### 1. 页面布局
采用经典数据大屏布局：
- 顶部：标题栏（项目名称 + 时间日期 + 实时数据刷新指示器）
- 左侧：两个图表面板（如趋势折线图 + 分类柱状图）
- 中间：核心区域（地图或大尺寸图表 + 关键数据指标卡片）
- 右侧：两个图表面板（如饼图/环形图 + 排行榜列表）
- 底部：滚动信息条或补充数据面板

### 2. 视觉风格
- 深色科技风主题，背景色 #0a0e27 或类似深蓝色系
- 边框采用发光边框效果（蓝色/青色霓虹光效）
- 数据卡片带渐变背景和微光动画
- 图表配色统一为科技蓝、青色、紫色系
- 标题使用渐变色文字

### 3. 功能要求
- 所有图表使用 ECharts 渲染，包含完整的 option 配置
- 使用 Pinia 管理全局状态（应用名称、主题、数据）
- 封装 useScreenScale composable 实现大屏自适应缩放
- 日期时间实时更新
- 数据模拟刷新效果（定时器模拟数据变化，图表动态更新）
- 数据卡片支持数字滚动动画（count-up 效果）

### 4. 数据模拟
使用 Mock 数据，覆盖以下场景：
- 月度销售趋势（折线图，12个月数据）
- 产品分类占比（环形图）
- 地区分布数据（柱状图或地图散点）
- 实时指标卡片（今日销售额、订单量、用户数、转化率）
- 排行榜数据（Top 10）

### 5. 代码规范
- 组件拆分合理，单文件职责清晰
- TypeScript 类型定义完整
- 无 ESLint 报错
- 所有样式使用 scoped SCSS

## 执行要求
1. 使用 Vite 创建项目（若已存在则在现有项目基础上开发）
2. 安装所需依赖（vue3、pinia、echarts、sass 等）
3. 按上述要求编写所有代码文件
4. 最终执行 `npm run dev` 启动开发服务器，并自动在浏览器中打开预览页面
